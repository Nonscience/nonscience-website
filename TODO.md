# 🚀 Nonscience 官网开发计划 (Roadmap)

本项目旨在构建一个极简主义（Swiss Style）的学术内容平台。目前首页已上线，后续开发将聚焦于响应式深度适配、全站页面补全及内容管理系统的集成。

---

## 📅 阶段一：响应式加固与交互打磨 (当前重心)
**目标**：消除设备适配死角，确保在中等屏幕（iPad）及各种移动端下的视觉美感。

- [ ] **[高] 中等屏幕 (768px~1024px) 深度适配**
    - **任务描述**：针对 iPad Air/Pro 等视口，调整 `.container` 宽度比例。
    - **备注**：解决三列网格在 iPad 上过于拥挤的问题，考虑切换为两列布局。
- [ ] **[中] 有机图形 (Blob) 随机化迭代**
    - **任务描述**：使用 CSS 8值 `border-radius` 进一步优化图形不规则感，增强艺术性。
- [ ] **[中] 移动端 Hero 区视觉增强**
    - **任务描述**：将 Hero 区高度提升至原来的 2 倍，并将主副标题间距扩大至 1.6 倍，增加呼吸感。

---

## 📂 阶段二：核心功能页面开发
**目标**：补全网站信息架构，实现从单页到多页站点的转型。

- [ ] **[高] 文章详情页 (Article Detail)**
    - **任务描述**：设计标准学术排版。包含：大标题、作者信息栏、双语摘要、正文（注意行高与字距）、参考文献引用块。
    - **备注**：需确保与首页风格高度一致。
- [ ] **[中] 文章列表页 (Archive)**
    - **任务描述**：展示所有历史稿件。复用首页的卡片逻辑，并增加基于栏目（Category）的标签筛选功能。
- [ ] **[中] 投稿页面 (Submission)**
    - **任务描述**：纯文字排版页面。详细列出投稿须知、版权协议，并设置显眼的按钮跳转至腾讯文档外部链接。

---

## 📢 阶段三：品牌与资讯补充
**目标**：完善平台背景信息，提升学术期刊的正式感。

- [ ] **[低] 公告页面 (Announcements)**
    - **任务描述**：列表式布局，用于发布征稿启事、获奖名单及期刊动态。
- [ ] **[低] 关于页面 (About Us)**
    - **任务描述**：介绍“非科学”的创刊理念、编辑团队及 Nonsense Studio 的愿景。
- [ ] **[低] 全站页脚同步**
    - **任务描述**：确保所有二级页面均包含已调优好的页脚。

---

## ⚙️ 阶段四：工程化与内容管理 (进阶)
**目标**：降低维护门槛，实现内容的自动化部署与管理。

- [ ] **[中] Netlify 部署与优化**
    - **任务描述**：将项目从 GitHub Pages 迁移或同步至 Netlify，利用其强大的 CDN 和自动化预览功能。
- [ ] **[低] 集成 Decap CMS**
    - **任务描述**：配置 `admin/config.yml`，为编辑提供可视化后台，实现无需改动代码即可管理文章和公告。

---

## 🛠️ 技术规范
1. **排版**：标题 `EB Garamond`，正文 `Inter`，中文 `Noto Sans SC`。
2. **响应式**：坚持使用 `clamp()` 动态字号，严禁二级页面出现横向滚动条。
3. **性能**：保持静态网站特性，首屏加载控制在 1.5s 内。
